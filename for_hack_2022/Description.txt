data_jobs.csv - Список вакансий 
	- JobId - уникальный идентификатор вакансии
	- Status - статус вакансии ( 1 - открытая, 2 - закрытая, 3 - приостановленная, 4 - отменена, 5 - черновик)
	- Name - название вакансии
	- Region - регион публикации вакансии 
	- Description - описание вакансии

data_candidates.csv - Данные о связи кандидатов, ваканси и статусов
	- CandidateId - уникальный id кандидата
	- Position - позиция указанная кандидатом
	- Sex - пол (2 - M, 1 - F, 0 - Undef)
	- Citizenship - гражданство
	- Age - возраст 
	- Salary - предполагаемая зп
	- Langs - языки
	- DriverLicense - категории прав 
	- Subway - Метро кандидата
	- Skills - описание навыков
	- Employment - Формат нагрузки (0 - 'Full', 1 - 'Part', 2 - 'Look-out', 3 - 'Change', 4 - 'Not full', 5 - 'does not matter')  
	- Shedule - Формат занятости (0 - 'Full', 1 - 'Flex', 2 - 'Project', 3 - 'Stage', 4 - 'Volunteering', 5 - 'does not matter')
	- CandidateRegion - регион кандидата
	- DateCreated - дата простановки статуса
	- JobId - id вакансии
	- CandidateStatusId - уникальный id статуса
	- Status - расшифровка статуса (приведены все существующий статусы для разных кампаний. Последовательность статусов на связи Кандидат-Вакансия-Статус определяет путь активности кандидата в рамках вакансии)

data_candidates_education.csv - Данные об образовании кандидатов
	- CandidateId - уникальный id кандидата
	- University - универчитет
	- Faculty - факутльтет
	- GraduateYear - год окончания
	
data_candidates_work_places.csv - Данные об опыте работы кандидатов
	- CandidateId - уникальный id кандидата
	- Position - должность 
	- FromYear - год начала работы
	- FromMonth - месяц начала работы
	- ToYear - год окончания работы
	- ToMonth - месяц окончания работы

Пример результирующего файла:
/result/result_job_#jobId.txt

Необходимый результат:
 Файлы по количеству вакансий в документе test/test_jobs.csv - в каждом список вакансий из документа test/test_candidates.csv отранжированные по параметру (от большего к меньшему), 
 который рассчитывается как коэффициент от 0 до 1 и является коэффициентом соответствия данного кандидата выбранной вакансии (1 - максимально подходит, 0 - максимально не подходит). 
 
 Как результат необходимо предоставить 10 файлов с название result_job_#jobId.txt, где #jobId - является id вакансии из файла test/test_jobs.csv. 
 Содержимое результирующих файлов должно быть в формате csv - первая колонка candidateId, вторая - коэффициент соответствия. 
 Каждый из результирующих файлов должен содержать количество строк равное количеству кандидатов из файлов test/test_candidates отсортированных по коэффициенту соответствия от большего к меньшему.
 Пример файла result/result_job_#jobId.txt.
 
 Также необходимо предоставить веб-интерфейс в котором реализована возможность ввода информации по вакансии и кандидату с возможностью получения коэффициента соответствия кандидата вакансии.
